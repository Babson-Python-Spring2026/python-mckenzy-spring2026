🧠 Can Minesweeper Be a 3D List?
Short answer: Yes, and here’s how it works conceptually:
Instead of:
      * board[row][col] = value
You’d have:
      * board[row][col][layer]
Where the third dimension (depth) stores multiple pieces of information about the same cell.
Example structure:
board[row][col] = [
   is_bomb,        # True/False
   adjacent_count, # 0–8
   is_revealed     # True/False
]
So:
      * board[0][1][0] → is there a bomb?
      * board[0][1][1] → how many bombs nearby?
      * board[0][1][2] → has the user revealed it?
________________


🔄 Why This Design Works
This replaces the need for:
      * separate “true board”
      * separate “visible board”
Everything is bundled into one structure, which is probably why your professor chose it.
👉 Tradeoff:
      * ✅ Compact, centralized state
      * ❌ Less readable than separate structures

def flood_fill(board, height, width, r, c):
   stack = [(r, c)]
  
   while stack:
       cr, cc = stack.pop()
      
       if not (0 <= cr < height and 0 <= cc < width):
           continue
      
       if board[cr][cc][2]:  # already revealed
           continue
      
       board[cr][cc][2] = True
      
       is_bomb, count, _ = board[cr][cc]
      
       if count == 0:
           for dr in [-1, 0, 1]:
               for dc in [-1, 0, 1]:
                   if dr != 0 or dc != 0:
                       stack.append((cr + dr, cc + dc))
